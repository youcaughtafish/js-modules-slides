# JavaScript Modules ### and why we need them Dylan Moore
> When looking at a larger program in its entirety, individual functions start to blend into the background. Such a program can be made more readable if we have a larger unit of organization. > <b>Modules</b> divide programs into clusters of code that, by some criterion, belong together. -- Marijn Haverbeke, [Eloquent JavaScript](http://eloquentjavascript.net/)
### Namespacing * JavaScript has no scoping between global and local (to a function) * Everything is either local to a function or visible everywhere
### Flat Projects ```js var chilling; function startChilling() { chilling = 'chilling'; } function chillingAndGrilling() { log(chilling+' & grilling'); } function dylanChillingAndGrilling() { log('dylan is..') startChilling(); chillingAndGrilling(); } ``` * related functionality problem: which code is needed for a specific piece of functionality * namespace problem: what's the interface? can I call anything I want and in any order? * dependency problem: are there external dependencies required for this code I need to include?
### Immediately-Invoked-Function-Expression ```js (function() { // code // tasty tasty // namespace'd code })(); ``` * declare a function that calls itself
### Basic JS Module ```js var Module = (function() { var privateMethod = function() {}; return { publicMethod: function() { // stuff // but also... privateMethod(); return true; } }; })(); var clientCode = Module(); clientCode.publicMethod(); // cannot call `clientCode.privateMethod()` ```
### Two 'Main Stream' Responses to JS Modules * AMD (Asynchronous Module Definition) * CommonJS
### AMD * Define a module ```js define('myModule', /* module name */ ['foo', 'bar'], /* dependencies */ function(foo, bar) { /* module definition */ function fooTheBar() { /* use foo and bar all you want */ } return { myModuleAwesomeMethod: fooTheBar } } ); ``` * Use a module ```js require(['myModule', 'jQuery'], function(myModule, $) { $('.cssClass').append(myModule.myModuleAwesomeMethod()); }); ```
### CommonJS * Module definition ```js var foo = require('foo'); var bar = require('bar'); function fooTheBar() { /* use foo and bar all you want */ } exports.myModule = { 'myModuleAwesomeMethod': fooTheBar }; ``` * Usage (client code) ```js var myModule = require('myModule'); var $ = require('jquery'); $('.cssClass').append(myModule.myModuleAwesomeMethod()); ```
### Node.js-Flavored CommonJS * `module.exports` instead of just `exports` ```js /* foo.js and bar.js modules in relative directories */ var foo = require('./foo'); var bar = require('../lib/bar'); // jquery module in 'node_modules' directory var $ = require('jquery'); function fooTheBar() { /* use foo and bar all you want */ /* also jquery, $('#all-the-things') */ } module.exports = { 'myModuleAwesomeMethod': fooTheBar }; ```
